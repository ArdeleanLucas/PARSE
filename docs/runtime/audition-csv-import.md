# Adobe Audition CSV speaker import

`POST /api/onboard/speaker` treats an uploaded CSV as an Adobe Audition marker export when concepts-style parsing (`id,concept_en`) returns no rows and the header contains `Name` and `Start` columns after case-insensitive trimming. Audition imports accept tab, comma, or semicolon delimiters, UTF-8 with or without BOM, and cue times in `H:MM:SS.mmm`, `M:SS.mmm`, or plain-seconds form. The `Name` field may use either a parenthesized hierarchical prefix such as `(8.4)- to listen to` or a plain integer prefix such as `9- nine`; terminal `A`/`B`/`C`/`D` variant markers are stripped from the label while the parsed id remains a string. On onboarding, valid Audition rows are appended in CSV row order to `tiers.concept.intervals` and mirrored to `tiers.ortho_words.intervals` with the CSV timestamps preserved exactly apart from the duration-derived end time fallback of `start + 1.0` when duration is missing or non-positive. This import intentionally does not populate or mutate `tiers.ortho`, `tiers.ipa`, or `tiers.bnd`; those remain downstream STT, IPA, and boundary-refinement outputs. Companion comments-file row-index joining is a follow-up and is not part of this contract.
